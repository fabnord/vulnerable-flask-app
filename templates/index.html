Server-Side Template Injection (SSTI) Demonstration

⚠️ Educational Purpose Only ⚠️

This application intentionally contains a vulnerability to demonstrate Server-Side Template Injection (SSTI). Never implement this code in a production environment.

What is SSTI?

Server-Side Template Injection occurs when user input is directly embedded into a template engine and then rendered. This can lead to code execution on the server if the input contains template expressions that are evaluated.

Vulnerable Implementation

This form passes user input directly to render_template_string() which evaluates it as a template.

Enter Template Code:

subprocess.Popen index: Loading...

Use this index for RCE payload:

{{ "''.__class__.__mro__[1].__subclasses__()[INDEX]('ls', shell=True, stdout=-1).communicate()[0].strip()" }}

Secure Implementation

This form safely passes user input as a parameter to a template, not as template code itself.

Enter Text:

The same input will be displayed as plain text, not evaluated.

Why is this Dangerous?

When template expressions are evaluated, they can:

  • Access sensitive configuration data
  • Read local files
  • Execute arbitrary code
  • Potentially gain full system access

Advanced Exploitation

In a real-world scenario, an attacker could use SSTI to execute arbitrary Python code by: 

{{ "''.__class__.__mro__[1].__subclasses__()[]('ls', shell=True, stdout=-1).communicate()[0].strip()" }}

Where <index> is the index of subprocess.Popen in the subclasses list. This would execute the 'ls' command and return its output.
